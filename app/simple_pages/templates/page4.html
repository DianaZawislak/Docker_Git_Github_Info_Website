How my project works?

This particular project uses two workflows to assure continuous integration and continuous delivery. It is production workflow and development workflow. Technically both are set up to do the same final thing, deploy the app. The difference is that one deploys to production and one to development. Before production can be updated, development must be checked, tested and merged. That is what triggers main app to update. Main app will update once new image is pushed to DockerHub for that to happen, merge has to be complete and all tests passed. DockerHub will receive new image via GitHub actions in workflow file and will push it to production to update main application.

Steps to set up the project?

  • 1. Create a new GitHub repository
  • 2. Create a new Docker repository in Dockerhub
  • 3. Create Heroku production app
  • 4. Create Heroku Development app
  • 5. Create API key within Heroku
  • 6. Add “secrets” to the GitHub
  • 7. Update info in prod.yml and dev.yml workflow files

As code is developed locally within PyCharm, commits are made to save and document changes of the application. Once committed, the application can be pushed to the working branch of GitHub. When a “pull” request is initiated, it will begin the Development workflow process. This starts with a test of code utilizing Pytest following the instructions contained in simple_pages_test.py. Once this passes, the workflow will move on to the deployment step where the code is sent to the development application in Heroku. The code can be reviewed for any errors and a “merge pull request” can be initiated to merge the branch into the main production branch. The “merge pull request” will then utilize the prod.yml workflow file and follow the GitHub Actions to again, test the code with Pytest. Once the test passes, the prod.yml workflow instructs GitHub to build the application and send it to the Docker repository. The final step is to deploy the working and tested product to the production application created within Heroku.
